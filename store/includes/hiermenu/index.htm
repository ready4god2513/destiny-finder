ColdFusion custom tags set: hiermenu 1.1

You can see a sample here

Download the tag

A set of three custom tags that display a minimalist, CSS based, menu. The output is fully XHTML complainant and works fine if served using an XHTML mime-type.

This menu is derived from the one created by Eric Meyer on his "More Eric Meyer on CSS" book. Eric used an .htc file to address IE's limitations, but I don't like that approach, so I went with a plain vanilla JavaScript solution for that. Mozilla and other Gekko based browsers do just fine with only the CSS, so the script filters IE using the document.all property.

The end results are quite similar to SuckerFish menus, but I realized it only at a very late stage (thanks Peter!), since I never looked very well at them before.

The whole look and feel is completely driven by the CSS file, so we will be able to create different menus (including horizontal ones), by just pointing to a different CSS file. Basically, every CSS can be a skin on its own. By the way, if you would like to share a custom skin, please contact me, I would be more than happy to include it inside the distribution.

Compared to others solutions, this kind on menus depends much less on JavaScript (for Mozilla JavaScript isn't required at all) and it's much easier to setup, especially for database driven menus. I's also worth to note that all the links are already available inside the XHTML code, so search engines and crawlers can easily follow them.

The menu has been tested with IE 5+ (Win), Mozilla 1.x and Safari 1.x

Requires ColdFusion Server 6.1 or more

Attributes (hiermenubar tag)

css Optional. The name of CSS file. Default to "flat_blue.css"
class Optional. The class attribute of the <div> tag. Default to "tmtHierbar"
id Optional. The id attribute of the <div> tag. Default to none
target Optional. This attribute affects all the child menu items, unless child tags overwrite it.
If you want to specify a target attribute for the whole menu you may consider using a base tag like: <base target="myTarget" />.
Default to none
width Optional. The width in pixel of menu. It's better practice to avoid using this attribute and define dimensions inside the CSS file instead. Default to none

Attributes (hiermenumenu tag)

label Optional. Text for the menu entry. Default to "Main Menu"
href Optional. The href attribute of the <a> tag. Default to "javascript:;"
class Optional. The class attribute of the <li> tag. Default to "tmtHiermenu"
id Optional. The id attribute of the <li> tag. Default to none
title Optional. The title attribute of the <a> tag. Default to none
target Optional. The target attribute of the <a> tag. This attribute also affects all the child menu items, unless child tags overwrite it.
If you want to specify a target attribute for the whole menu you may consider using a base tag like: <base target="myTarget" />.
Default to none

Attributes (hiermenuitem tag)

label Optional. Text for the menu item. Default to "Sub Menu"
href Optional. The href attribute of the <a> tag. Default to "javascript:;"
class Optional. The class attribute of the <li> tag. Default to none
id Optional. The id attribute of the <li> tag. Default to none
title Optional. The title attribute of the <a> tag. Default to none
target Optional. The target attribute of the <a> tag. Default to none
