CF_IADropdown

IADropdown is a cold fusion custom tag developed to counter the often tedious task of including the various kinds of dropdown lists in applications. Not only does IADropdown make it easy to add dropdowns, it makes it easier to control how they are displayed and which items are highlighted.

IADropdown has 6 main dropdown modes, and 3 companion modes that help format submitted date/time information.

  • Numbered, for creating numeric dropdowns
  • Combolist, for creating key/value dropdowns
  • Valuelist, for converting lists into dropdowns
  • Boolean, for yes/no dropdowns
  • Time, for time dropdowns (hour/minute/AM-PM)
  • Date, for date dropdowns (month/date/year)

…and…

  • Assemble_time, to create a valid time object out of the submitted time dropdown
  • Assemble_date, to create a valid date object out of the submitted date dropdown
  • Assemble_datetime, to create a valid date/time object out of the submitted date and time dropdown

If an error is detected in any of the modes, the variable error_message will be returned.

Install

Installation is straight-forward. Simply place the IADropdown.cfm file in your Cold Fusion Server custom tags directory, i.e.:

C:\Cfusion\CustomTags\IADropdown.cfm

IADropdown Modes

MODE: numbered

Creates a dropdown number list. Must be enclosed in a <SELECT></SELECT> HTML statement.

<CF_IADropdown
mode = "numbered"
from = "start number"
to = "end number"
step = "step value"
selected = "selected number">

ATTRIBUTES

from: a numeric value representing the first number to appear in the drop-down

to: a numeric value represensting the last number to appear in the drop-down

step: a numeric value that sets the value the number list is incremented/decrimented

selected: the number that will be marked as "selected" in the drop-down

EXAMPLE

<CF_IADropdown
mode = "numbered"
from = "100"
to = "1000"
step = "100"
selected = "400">

MODE: combolist

Creates a dropdown list with separate keys (displayed in the list) and values (which are submitted). Must be enclosed in a <SELECT></SELECT> HTML statement.

<CF_IADropdown
mode = "combolist"
valuelist = "a list of keys and values"
selected = "selected value">

ATTRIBUTES

valuelist: a list with a combination of the displayed "key" and the associated "value", separated by a bar. For instance, one|1 would create the HTML

selected: the value that will be marked as "selected" in the drop-down

EXAMPLE

<CF_IADropdown
mode = "combolist"
valuelist = "one|1,two|2,three|3,four|4"
selected = "4">

MODE: valuelist

Creates a dropdown list of values. Must be enclosed in a <SELECT></SELECT> HTML statement.

<CF_IADropdown
mode = "valuelist"
valuelist = "a list of values"
selected = "selected value">

ATTRIBUTES

valuelist: a list of values.

selected: the value that will be marked as "selected" in the drop-down

EXAMPLE

<CF_IADropdown
mode = "combolist"
valuelist = "one,two,three,four"
selected = "three">

MODE: boolean

Creates a yes/no dropdown list. Must be enclosed in a <SELECT></SELECT> HTML statement.

<CF_IADropdown
mode = "boolean"
selected = "selected value">

ATTRIBUTES

selected: the value that will be marked as "selected" in the drop-down (either 0 or 1)

EXAMPLE

<CF_IADropdown
mode = "boolean"
selected = "1">

MODE: time

Creates a combination drop-down, with hours, minutes, and AM/PM.

<CF_IADropdown
mode = "time"
label = "field label prefix"
set_time = "time to display as current"
hour = "hour to set"
minute = "minute to set"
period = "AM/PM"
step = "minute step value"
add_minutes="number of minutes to add to set time">

ATTRIBUTES

label: a prefix attached to each of the hour/minute/period dropdowns. For instance, if the label is "ctime", then there would be a form element called "ctime_hour", "ctime_minute", and "ctime_period".

set_time: optional; the time to be displayed by the drop-downs.

hour: optional; the hour to be displayed by the hour drop-down; overrides set_time, must be used with minute and period.

minute: optional; the minute to be displayed by the minute drop-down; overrides set_time, must be used with hour and period.

period: optional; the AM/PM marker to be displayed by the period drop-down; overrides set_time, must be used with hour and minute.

step: optional; the step value for the minutes. Default is 15.

Add_minutes: resets the time by adding the numeric value assigned to this attribute to the current time.

EXAMPLE

<CF_IADropdown
mode = "time"
label = "ctime"
set_time = "#now()#"
step = "1">

MODE: assemble_time

Assembles timestamp based upon hour/minute/period.

<CF_IADropdown
mode = "assemble_time"
hour = "numeric hour value"
minute = "numeric minute value"
period = "AM/PM">

hour: the hour of the time to be created.

minute: the of the time to be created.

period: the AM/PM marker of the time to be created.

RETURNS: assembled_time

EXAMPLE

<CF_IADropdown
mode = "assemble_time"
hour = "12"
minute = "30"
period = "PM">

MODE: date

Creates a combination drop-down, with month, date and year.

<CF_IADropdown
mode = "date"
label = "field label prefix"
set_date = "date to display as current"
year = "year to set"
month = "month to set (numeric value 1-12)"
date = "day of month to set"
first_of_month = "sets dropdown to start on the first day of the month"
start_year = "first year to be displayed"
year_range = "number of years to display beyond displayed year"
add_months="number of months to add to set date">

ATTRIBUTES

label: a prefix attached to each of the month/date/year dropdowns. For instance, if the label is "cdate", then there would be a form element called "cdate_month", "cdate_date", and "cdate_year".

set_date: optional; the date to be displayed by the drop-downs.

month: optional; the month to be displayed by the month drop-down; overrides set_date.

date: optional; the day of month to be displayed by the date drop-down; overrides set_date.

year: optional; the year to be displayed by the year drop-down; overrides set_ date.

first_of_month: optiona;l resets the day of the month set to be displayed to the first.

start_year: optional; sets the first year to be displayed.

year_range: optional; the number of years to be added to the year being displayed.

add_months: optional; resets the date by adding the numeric value assigned to this attribute to the current date.

EXAMPLE

<CF_IADropdown
mode = "date"
label = "cdate"
year = "2000"
month = "6"
date = "23"
year_range="20">

MODE: assemble_date

Assembles timestamp based upon date/month/year.

<CF_IADropdown

mode = "assemble_date"

year = "numeric year value"

month = "numeric month value"

date = "numeric day of month">

month: the month of the date to be created.

date: the day of month of the date to be created.

year: the year of the date to be created.

RETURNS: assembled_date

EXAMPLE

<CF_IADropdown
mode = "assemble_date"
year = "1995"
month = "04"
date = "20">

MODE: assemble_datetime

Assembles timestamp based upon date/month/year.

<CF_IADropdown
mode = "assemble_datetime"
hour = "numeric hour value"
minute = "numeric minute value"
period = "AM/PM"
year = "numeric year value"
month = "numeric month value"
date = "numeric day of month">

month: the month of the date to be created.

date: the day of month of the date to be created.

year: the year of the date to be created.

RETURNS: assembled_datetime

EXAMPLE

<CF_IADropdown
mode = "assemble_date"
year = "1995"
month = "04"
date = "20"
hour = "12"
minute = "30"
period = "PM">
